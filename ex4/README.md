# ex4の課題（前処理の変更と既製モデルによる試合分析）

## 課題の概要

ノートブックの前処理は，Wyscout の5大リーグのうち **England（プレミアリーグ 2017/18）以外を削除**しており，デフォルトでは England のデータだけが使われる．

本課題では，前処理の処理内容を理解して **England 以外のリーグ（例：La Liga）を残すように変更**し，そのリーグから1試合を選んで，**配布済みの学習済みモデル（自分では学習しない）** で推論し，その試合を可視化・分析する．
学習はせず，「前処理を変えて別リーグのデータを取り出し，用意されたモデルで推論する」流れを通して，前処理パッケージとモデル推論の使い方を理解することを目的とする．

対象ノートブック：

- `5.3_5.4_Soccer_Event_Preprocess_Modeling_OpenSTARLab.ipynb`

> 本課題で選んだ「リーグ・試合」は，ex5 で同じものを再利用する．

## 課題

1. 前処理を変更し，England 以外の1リーグ（例：La Liga＝Spain）を残して再前処理する
2. そのリーグから1試合を選ぶ（例：レアル・マドリード vs バルセロナ＝クラシコ）
3. 配布済みの学習済みモデルで，その試合に対して推論する
4. その試合の予測結果（イベント間時間・ゾーン・アクション）や HPUS を可視化し，分析・考察する

## 課題の進め方

1. ノートブックの処理を確認する
   - データダウンロードのセルで，`events_Spain.json` などを `rm` で削除している箇所を読む
   - どのファイルを残せば目的のリーグが使えるか理解する（La Liga なら Spain を残す）

2. 課題1：前処理の変更
   - 削除処理を変更し，England 以外の1リーグ（例：Spain＝La Liga）を残す
   - 残したデータで `Event_data(... preprocess_method="UIED" ...)` を再実行し，`data.csv` を出力する
   - 必須列（`action`, `start_x`, `start_y`, `seconds`, `delta_T`, `team`, `match_id` など）と欠損の有無を確認する

3. 課題2：1試合のピックアップ
   - `match_id` を1つ選ぶ（どの試合か，対戦カードが分かるように明記する）
   - 例：La Liga ならクラシコ（レアル・マドリード vs バルセロナ）など，特徴のある試合を選ぶとよい
   - 選んだ試合のイベントだけを抽出する

4. 課題3：学習済みモデルによる推論
   - ノートブックの手順で配布済みモデル（`_model_23.pth` など）をダウンロードする
   - 選んだ試合（または当該リーグ）を入力として `inference()` を実行し，`loss_df` と予測結果を得る
   - **自分でモデルを学習しないこと**（あくまで「用意されたモデル」を使う）

5. 課題4：可視化分析と考察
   - 以下のような観点で，少なくとも3種類以上の図を作成する
     - アクション種別（`action`）の出現頻度
     - アクション開始位置（`start_x`, `start_y`）のピッチ上ヒートマップ／散布図
     - 予測イベントと実イベントの比較（当たっている／外れている箇所が分かるように）
     - その試合の HPUS / HPUS+
   - 図と `loss_df` をもとに，この試合でモデルがどの程度予測できているかを考察する

6. 発表に向けて整理する
   - 選んだリーグ・試合と，前処理の変更点を説明する
   - 試合の可視化から分かった特徴を説明する
   - 用意されたモデルがこの試合をどの程度予測できたか，図と数値の両方を使って考察する

## レポートに含める内容

1. 課題1：前処理の変更
   - 残したリーグと，変更した処理の説明
   - 再前処理した `data.csv` の概要（試合数・行数など）

2. 課題2・3：試合の選択と推論
   - 選んだ `match_id`（対戦カードを明記）
   - 推論に使った設定と `loss_df`

3. 課題4：可視化分析
   - 作成した図（最低3種）
   - 予測 vs 実イベントの比較図
   - HPUS / HPUS+ の図
   - この試合でモデルがどの程度予測できたかの考察

## ヒント

- Python を使う場合
  - データ処理：`numpy`, `pandas`
  - 描画：`matplotlib`（必要に応じてサッカーピッチ描画ライブラリ）
- 残せるリーグのファイル名：`events_Spain.json`（La Liga），`events_Italy.json`（Serie A），`events_France.json`（Ligue 1），`events_Germany.json`（Bundesliga）と対応する `matches_*.json`
- 配布済みモデルとあわせて `min_max_dict.json`（正規化用）・`hyperparameters.json` もダウンロードされる．推論にはこれらが必要
- ヘッドレス環境で実行する場合は，図は `plt.savefig(...)` でファイルに保存すること

## 注意

- Colab で実行する場合は，ランタイムを GPU に設定すること（推論のみなら CPU でも可だが GPU 推奨）
- ノートブック内のパスは `/content/` を前提としているため，ローカル実行する場合はパスを修正すること
- データのダウンロードが失敗する場合は，ノートブックの案内に従って figshare から手動配置すること
- 本課題では**モデルの学習は行わない**（自前学習は ex5 で扱う）
- グラフの軸ラベル，凡例，対象とした `match_id` を明記すること
- 数値（`loss_df`）だけでなく，可視化結果を用いて考察すること
- 課題4まで到達できなくても，まずは課題1〜3（前処理変更・試合選択・推論）を完成させること
