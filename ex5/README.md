# ex5の課題（自前モデルの学習と同一試合での比較分析）

## 課題の概要

ex4 では，配布済みの学習済みモデルを使って，別リーグの1試合を推論・可視化した（学習はしなかった）．
本課題では，`openstarlab-event` パッケージを用いて **自分でモデルを学習**し，**ex4 と同じ試合**に適用して可視化・分析する．
モデルの種類・使うデータ・特徴量・パラメータを自分で選んで学習し，ex4 で用いた配布済みモデルと結果がどう変わるかを比較・考察することを目的とする．

対象ノートブック：

- `5.3_5.4_Soccer_Event_Preprocess_Modeling_OpenSTARLab.ipynb`（主にステップ 5〜8：YAML設定・学習・推論・分析）

> ex4 で選んだ**リーグ・試合**，および ex4 で得た**配布済みモデルの推論結果**を，本課題の比較対象として再利用すること．

## 課題

1. 学習設定を自分で決める（次のような変更を1つ以上行う）
   - 使うモデルを変える（`NMSTPP` / `LEM` / `Seq2Event` / `FMS` など）
   - 使うデータを変える（England だけでなく複数リーグを入れる など）
   - 特徴量を変える（`use_other_features`，使う特徴量の構成 など）
   - パラメータを変える（`num_epoch`，`seq_len`，学習率，モデル次元 など）
2. 自分でモデルを学習する
3. ex4 と同じ試合に対して，学習したモデルで推論する
4. その試合を可視化・分析し，ex4（配布済みモデル）の結果と比較・考察する
5. （発展）複数の設定で学習し，どの設定が良かったかを比較する

## 課題の進め方

1. ノートブックの処理を確認する
   - 学習用 YAML の主要パラメータ（`train_path`, `valid_path`, `save_path`, `num_epoch`, モデル種別，`seq_len`，`use_other_features` など）を確認する
   - `Event_Model(...)` で学習し，`inference()` で `loss_df` が得られる流れを確認する

2. 課題1：学習設定の決定
   - 上記の変更点から，自分で何を変えるかを決める（複数組み合わせてよい）
   - なぜその設定にしたか，狙いを言語化しておく

3. 課題2：自前モデルの学習
   - 決めた設定で前処理・YAML設定・学習を行う
   - **Colab の実行時間を考慮し，`num_epoch` は小さめ（例：数エポック）から始める**
   - `Seq2Event` は重い．Colab では `NMSTPP` / `LEM` / `FMS` が比較的軽い

4. 課題3：同一試合への推論
   - ex4 で選んだ**同じ `match_id`** に対して，学習したモデルで推論する
   - 比較を公平にするため，入力する試合・評価指標は ex4 と揃える

5. 課題4：可視化分析と比較・考察
   - ex4 と同じ観点で，自前モデルの予測を可視化する
     - 予測イベント vs 実イベント（当たり外れ）
     - その試合の HPUS / HPUS+
   - ex4（配布済みモデル）の結果と並べて比較する
     - `loss_df` の指標を**比較表**にまとめる（最良値を太字，スケール統一）
     - 予測図・HPUS図を重ねる／並べる
   - 自前モデルは配布済みモデルと比べてどう変わったか，なぜそうなったかを考察する

6. 課題5（発展）
   - 設定を複数試して比較する（例：モデルを2種，使うリーグを変える，`seq_len` を変える など）
   - どの設定が良かったか，図と数値の両方で説明する
   - 改善しなかった設定も1つ取り上げ，なぜ改善しなかったか考察する

7. 発表に向けて整理する
   - 自前モデルの学習設定（モデル・データ・特徴量・パラメータ）と，その狙いを説明する
   - 配布済みモデルと自前モデルの比較結果を，比較表と図で示す
   - 同じ試合で結果がどう変わったか，その理由を考察する
   - 評価値と可視化結果の両方を使って考察する

## レポートに含める内容

1. 課題1・2：自前モデルの学習
   - 変更した設定（モデル／データ／特徴量／パラメータ）とその狙い
   - 学習に使ったデータと主な設定（`num_epoch` 等）

2. 課題3・4：同一試合での比較
   - 対象 `match_id`（ex4 と同一であること）
   - 配布済みモデル vs 自前モデルの比較表（`loss_df`）
   - 予測 vs 実イベントの比較図
   - HPUS / HPUS+ の比較図
   - 結果がどう変わったか，その理由の考察

3. 課題5：発展（取り組んだもののみ）
   - 比較した設定と結果
   - 良かった設定・改善しなかった設定とその理由

## ヒント

- Python を使う場合
  - データ処理：`numpy`, `pandas`
  - 描画：`matplotlib`
  - 最適化（発展）：`Optuna`
- モデルは `Event_Model('<モデル名>', '<YAMLパス>')` で切り替えられる．YAML は openstarlab の `Event` リポジトリの `model_yaml/` から取得できる（`train_LEM.yaml`，`train_Seq2Event.yaml` など）
- 全モデルで `basic_features` / `other_features` / `num_actions` は共通なので，前処理した同じ `data.csv` をそのまま使える
- 複数リーグを使う場合は，ex4 で見た前処理の削除セルを調整し，残すリーグを増やす
- 配布済み pretrained は1種類のみ．他モデルや他設定は必ず自前学習が必要（＝少エポック前提）
- 公平に比較するため，比較対象以外（対象試合，評価指標）は固定すること
- ヘッドレス環境で実行する場合は，図は `plt.savefig(...)` でファイルに保存すること

## 注意

- Colab で実行する場合は，ランタイムを GPU に設定すること
- ノートブック内のパスは `/content/` を前提としているため，ローカル実行する場合はパスを修正すること
- 設定を変えて再学習する前に，学習済みモデル・`loss_df`・YAML を保存しておくこと
- 比較表は指標のスケールを統一し，対象としたモデル・設定・`match_id` を明記すること
- 評価値だけでなく，可視化や考察を必ず含めること
- ex4 と同じ試合・同じ評価条件で比較すること（比較が成立しなくなるため）
- 発展課題まで到達できなくても，まずは課題1〜4（自前学習・同一試合推論・配布済みとの比較・考察）を完成させること
