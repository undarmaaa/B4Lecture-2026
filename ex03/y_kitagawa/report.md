# トラッキング性能の比較実験と考察

## 考察

表にまとめた結果から、以下のように考察する。

* **最も良かった組み合わせ**
  **`YOLO11x`** と **`ByteTrack`** の組み合わせが最も優れており、 TI-HOTA (0.7440) をはじめ、すべての評価指標で最高スコアを記録している。

* **検出器のモデルサイズによる変化**
  検出器を軽量な `YOLO11n` から大規模な `YOLO11x` へと大きくすることで、性能は大きく改善した。TI-HOTA のスコアが 0.16〜0.25 程度から 0.74 前後まで跳ね上がっており、検出モデルの精度が追跡全体のパフォーマンスに直結していることがわかる。

* **追跡器の変更による影響**
  追跡器を `BoT-SORT` から `ByteTrack` に変更することで、指標が向上する傾向が見られた。特に `YOLO11n` を使用した場合、TI-HOTA が 0.1608 から 0.2582 へと大きく改善している。
  一方で `YOLO11x` を用いた場合、BoT-SORTとByteTrackの間で評価指標に大きな差が見られなかった。これは、 `YOLO11x` の検出精度が非常に高く、毎フレーム安定して正確なバウンディングボックスが出力されるため、高度な予測を行わずとも単純な位置の重なり（IoU）だけで容易にIDを維持できたためと考えられる。
  ByteTrackの強みは「信頼度スコアの低い検出結果も破棄せずに追跡のヒントとして再利用する」点にある。 `YOLO11n` のような軽量モデルではこの強みがわかりやすく発揮されたが、 `YOLO11x` ではそもそも低スコアの検出自体がほとんど発生しないため、両者の追跡アルゴリズム間で性能差がつきにくかったと推測される。

* **検出性能とID維持性能、どちらが課題か**
  すべての実験条件において、ID維持性能を示す `TI-AssA` よりも、検出性能を示す `TI-DetA` の値が低くなっている。このことから、追跡の途中でIDが変わってしまうことよりも、見逃しや誤検出といった人物の正確な検出の方が大きな課題になっていると考えられる。
